🛡️ Multi-tenant Authentication Flow

JWT 기반 로그인, 강제 로그아웃을 위한 TokenVersion 검증, 그리고 모든 데이터 쿼리에 동적으로 주입되는 gym_id 테넌트 격리 로직입니다.

sequenceDiagram actor User as Client App participant Auth as Security Filter participant API as Services participant DB as MySQL (JPA) User->>Auth: POST /api/auth/login (Email/PW) Auth->>DB: Validate Credentials DB-->>Auth: Success (Returns gym_id & token_version) Auth-->>User: Access + Refresh Token (JWT) Note over User,Auth: JWT Payload: { sub: email, gymId: 1, role: MEMBER, version: 2 } User->>Auth: GET /api/members/me (Bearer Token) Auth->>Auth: Validate Signature & Expiration Auth->>DB: Check TokenVersion (If changed -> Force Logout) Auth->>API: Pass Request with Principal (gymId context) rect rgb(20, 30, 50) Note over API,DB: Hibernate @Filter applies WHERE gym_id = ? to ALL queries API->>DB: SELECT * FROM member DB-->>API: Filtered Tenant Data (gym_id = 1) end API-->>User: 200 OK + Data

Frontend Architecture Highlights

  • App Router (Next.js 16): Server Components(RSC)와 Client Components를 전략적으로 혼합하여 렌더링 성능 극대화
  • PWA / Service Worker: 웹앱을 네이티브 앱처럼 사용할 수 있도록 오프라인 캐싱 및 백그라운드 FCM Push 알림 수신
  • Tailwind CSS 4 + shadcn/ui: 일관된 디자인 시스템 적용 및 높은 접근성(a11y) 확보
  • Custom DatePicker: 무거운 서드파티 라이브러리를 배제하고 캘린더/주간 뷰를 독자 구현하여 성능 최적화
  • Security (DOMPurify): 관리자가 리치 에디터로 작성한 '오늘의 운동' HTML을 클라이언트 사이드에서 XSS 필터링 후 안전하게 렌더링
